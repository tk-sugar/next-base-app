# AI_RULES.md

AIがこのプロジェクトのコードを生成・編集する際に従うルール。

## 基本方針

- 要求された変更のみ行い、不必要なリファクタリングや機能追加はしない
- シンプルさを優先する。過度な抽象化・汎用化はしない
- 既存のコードパターンと一貫性を保つ

## 言語 / フレームワーク

### TypeScript
- `strict: true` を維持する
- `any` は使わない。型が不明な場合は `unknown` を使い適切にナローイングする
- 型定義は `type` を優先し、拡張が必要な場合のみ `interface` を使う

### Next.js
- **App Router** のみ使用する。`pages/` ディレクトリは作らない
- データフェッチはサーバーコンポーネントで行うことを優先する
- `"use client"` は本当に必要な場合（ブラウザ API、イベントハンドラ等）のみ使う
- パスエイリアス `@/*` を使う（`../../` の相対パスは避ける）

### Tailwind CSS
- バージョン 4 の記法を使う（`globals.css` に `@import "tailwindcss"` のみ）
- `tailwind.config` ファイルは作らない（v4 ではデフォルト不要）

### データベース
- `src/lib/db.ts` の Pool を使う
- SQL はプレースホルダー（`$1`, `$2`）でパラメータ化する。文字列結合は絶対にしない
- 例: `pool.query('SELECT * FROM users WHERE id = $1', [id])`

## ファイル構成

```
src/app/          # ページ・レイアウト・API Route
src/lib/          # 共通ユーティリティ（DB等）
src/components/   # 共通コンポーネント（必要になったら作成）
src/types/        # 型定義（必要になったら作成）
```

## コードスタイル

- 関数コンポーネントのみ使う（クラスコンポーネントは使わない）
- `export default` はページ・レイアウト等の Next.js 規約に従うファイルのみ
- それ以外は named export を使う
- コメントは自明でないロジックにのみ書く

## セキュリティ

- SQL インジェクション防止: 必ずパラメータ化クエリを使う
- 環境変数: シークレットは `.env.local` に置き、コードに直接書かない
- `NEXT_PUBLIC_` プレフィックスはクライアントに公開してよい変数のみに付ける

## パッケージ管理

- **npm** を使う（bun・yarn・pnpm は使わない）
- 新しいパッケージを追加する前に、既存の方法で解決できないか検討する

## Docker

- Node.js は `24-alpine` を使う
- 本番 Dockerfile はマルチステージビルドを維持する
- `next.config.ts` の `output: "standalone"` は削除しない
